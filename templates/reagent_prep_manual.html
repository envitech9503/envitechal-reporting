About this manual

This guide explains every part of the Reagent & Standards Preparation module of the Envi Tech AL LIMS, used by the Karachi and Lahore laboratories to record reagent and standard preparations, standardise titrants, verify records and generate controlled PDF logs.

Use the search box above to jump to any topic, or the category chips to narrow the list. Press / anywhere to focus the search box. Sections marked Supervisor / QA concern verification, the audit trail and controls; everything else is day-to-day analyst use. The module gives full access to all logged-in users — actions are open, and the system records who did what and when for traceability.

Getting started Opening the module

The module lives inside the QC area of the LIMS. You can reach it in two ways:

  • From the top navigation bar, open QC and choose Reagent & Standards Prep.
  • Directly by address: the entry form is at /qc/reagent_prep/ and the records list is at /qc/reagent-prep-list/.

The module has two working areas that you will move between constantly:

  • The form — where you create, edit, clone or view a single preparation record.
  • The list — where you search, filter, sort and print existing records.
You must be logged in. Every logged-in user has full access to create, edit, verify and print; the system records your name against the actions you take.

Getting started A quick tour of the entry form

Working down the entry form you will find, in order:

  1. Header details — location, reagent type, reagent name and number, month, dates, volume and concentration.
  2. Chemicals used — a table of the chemicals that went into the preparation, with live inventory recall.
  3. Standardisation & Correction Factor — appears only for Standardised titrant types; computes the true strength and correction factor.
  4. Standardisation History — a read-only running log of every standardisation of that titrant (appears once at least one event exists).
  5. Description / Method of Preparation — the narrative method, plus Prepared By, Verified By and Remarks.
  6. Calculator — an embedded, fully-loaded preparation calculator for weigh-and-dissolve, dilutions, calibration series and conversions.

Above the form sit the action buttons — Save, Save & New, Verify, History and the verification badge.

Data entry Creating a new preparation

Open the form at QC → Reagent & Standards Prep → New Preparation and work through the fields:

  1. Choose the Location (Karachi or Lahore).
  2. Pick the Reagent Type — this decides whether the standardisation panel appears.
  3. Enter the Reagent Name (required) and a Reagent No. (the system suggests the next number for you).
  4. Set the D.O.P. (date of preparation) and D.O.E. (date of expiry).
  5. Add the chemicals used, one row per chemical.
  6. For a titrant, complete the Standardisation panel.
  7. Write the method, set Prepared By, and add any Remarks.
  8. Click Save (or Save & New to store this one and immediately start another).
The Reagent Name is the only strictly required field — saving without it is blocked. Everything else can be completed progressively, but a complete record is expected for regulatory logs.

Data entry Reagent type

The Reagent Type classifies the preparation and controls the form. The available types are: Reagent, Standard, Standardised titrant, Buffer and % solution.

Choosing Standardised titrant reveals the Standardisation & Correction Factor panel and, once events exist, the Standardisation History table. Any other type hides both.

If you change a titrant to another type and save, its stored standardisation record is removed (the reagent is no longer a titrant). See Editing a titrant safely for exactly when the factor is kept versus cleared.

Data entry Reagent number & auto-suggestion

The Reagent No. field helps you keep a consistent, per-laboratory numbering scheme.

  • As you focus the field, the system suggests the next number for each prefix already used in that lab (for example, if RG-12 is the highest, it offers RG-13). The suggestion is advisory — you can type any value.
  • If the number you enter already exists in the same lab, the system shows a duplicate warning when you save and asks you to confirm. Choose Save anyway to proceed, or change the number.
Suggestions are calculated per laboratory, so Karachi and Lahore numbering do not interfere with each other.

Data entry Dates: D.O.P., D.O.E. and month

D.O.P. is the date the reagent was prepared; D.O.E. is its expiry date. The Month field groups records for the monthly printed log and can be set with the month picker.

The system will not let the expiry (D.O.E.) fall before the preparation date (D.O.P.). If it does, the save is rejected with the message “D.O.E. (expiry) cannot be earlier than D.O.P. (prepared)”. Correct the dates and save again.

Dates display in dd-mm-yyyy format throughout the module and the printed logs.

Data entry Chemicals used (with inventory recall)

Record each chemical that went into the preparation on its own row of the chemicals table. Click Add Row for more rows.

Start typing a chemical name and the system offers matches from live inventory. Selecting one recalls the CAT No., LOT No., Make and Expiry automatically. You can still edit any field afterwards.

Data entry Volume, concentration & description

Final volume records the total made-up volume; Concentration value and unit record the nominal strength of the preparation (for example 0.1 / N).

The Description / Method of Preparation box is a free-text narrative — describe the weighing, dissolution, dilution, standardisation and storage steps. Below it, Prepared By, Verified By and Remarks capture the people and any notes. Prepared By defaults to your name on a new record.

Data entry Live preview & the calculator

As you type, a live preview mirrors how the controlled PDF will look. It is presentation only — nothing is stored until you click Save.

The embedded Fully-Loaded Preparation Calculator handles weigh-and-dissolve, dilutions, calibration series, standardisation, % solutions and unit converters. It is a working aid; values are not written to the record automatically — carry the results you need into the fields yourself.

Data entry Saving — Save vs Save & New

  • Save stores the record and keeps you on it (useful before verifying or printing).
  • Save & New stores the current record and immediately opens a fresh blank form — ideal for entering many preparations in one sitting.
If you try to leave the form with unsaved changes, the browser warns you first, so work is not lost by accident.

Standardisation Standardisation & correction factor

For a Standardised titrant, complete the standardisation panel to compute the titrant’s true strength and correction factor. Enter the primary standard, its mass, purity and equivalent/molar weight, the titrant volume used, and the nominal strength; the system computes the rest.

FieldMeaning
Primary standardThe reference substance (e.g. CaCO₃, KHP, K₂Cr₂O₇).
PS mass / purity / eq.wtMass weighed (g), purity (%) and equivalent or molar weight.
Titrant used (mL)Volume of titrant delivered to the endpoint.
Nominal strength / unitThe labelled strength (e.g. 0.1) and unit (N or M).
True strength computedCalculated actual strength of the titrant.
Correction factor computedTrue strength ÷ nominal strength.
Std date / Next dueDate standardised and the date the next standardisation is due.
Formula used: True strength = (mass × purity/100) ÷ (eq.wt × titrant in litres), and factor = true ÷ nominal. The record is stored against the selected location.

Standardisation Standardisation History new

Titrants are re-standardised periodically over their life. The Standardisation History section keeps a read-only running log of every standardisation and re-standardisation of that titrant, so you can see how its correction factor has changed over time.

Each time you save the titrant with a standardisation whose values differ from the last logged one, a new dated entry is added automatically. The table shows, newest first:

ColumnShows
Std dateThe date that standardisation was performed.
Primary stdThe primary standard used.
True strength / FactorThe computed true strength and correction factor for that event.
Nominal / Titrant (mL)Nominal strength and titrant volume used.
Next dueWhen the next standardisation was scheduled.
Recorded by / atThe user who logged it and the timestamp.
The log is append-only and read-only — you cannot edit past entries, which preserves the traceable history. If you save the titrant again without changing the standardisation values, no duplicate entry is added. The table appears only when at least one standardisation event has been logged.

Standardisation Editing a titrant safely

You can edit a titrant record — for example to correct a name or add a chemical — without losing its standardisation factor. When you save a titrant and leave the standardisation values unchanged, the stored factor is kept exactly as it was.

  • Re-standardising — enter new standardisation values and save: the current factor updates and a new history entry is logged.
  • Editing other details — change the name, chemicals or method and save: the standardisation factor is preserved and no history entry is added.
  • Changing the type away from titrant — the standardisation record is removed, because the reagent is no longer a titrant.
This protection means routine edits never accidentally erase a titrant’s correction factor.

Verification & audit Supervisor / QA Verifying a record

Use Verify to mark a saved record as checked. The button is open to any logged-in user; the system records who verified it and when, and shows a green Verified by … · date badge on the record.

  • Click Verify to stamp the record; the label switches to Unverify so the action can be reversed.
  • A record must be saved first before it can be verified.
If a verified record is edited and saved again, its verification is automatically cleared — a changed record must be re-checked. The clearing is logged in the record history.

Traceability: verification stores the verifier’s identity and timestamp even though the action is open to everyone.

Verification & audit Supervisor / QA Record history (audit trail)

Click History on a saved record to open its audit trail — an append-only log of everything that has happened to the record. Each entry shows the time, the user, the action and a summary of what changed.

Logged actions include: created, updated (with a field-by-field summary of changes), type-changed, verified and unverified.

The audit trail is read-only and cannot be altered, giving a complete, defensible record of the reagent’s life for inspection by regulatory bodies (Sindh EPA, Punjab EPA).

The audit trail (who did what to the record) is distinct from the Standardisation History (the running log of standardisation events). Both are read-only.

Verification & audit Concurrent-edit protection

If two people open the same record and both save, the module prevents one from silently overwriting the other. When you save a record that someone else changed while you were editing, the save is blocked with a message like “This record was changed by … while you were editing. Reload to get the latest version before saving.”

Reload the record to pick up the latest version, re-apply your change, and save again.

Verification & audit Read-only View mode

The View action (on the list) opens a record in read-only mode. A banner makes clear you are viewing, not editing, and the fields are locked. This is ideal for checking a record or showing it to a reviewer without any risk of accidental change. You can still open its PDF from View mode.

List & search The records list

The list at QC → Reagent & Standards Prep shows all preparation records with their number, name, type, location, D.O.P., D.O.E., status and actions. From here you search, filter, sort, open, clone and print records.

Each row’s actions are: View (read-only), Edit, Clone and PDF.

List & search Filtering & searching

Use the toolbar above the list to narrow it down:

ControlEffect

StatusAll, Active, Expiring (within 14 days) or Expired.

The Status column shows a coloured pill — Expired (red), Expiring (amber) or Active (green) — plus a green tick for verified records, so you can spot reagents needing attention at a glance.

Use the Expiring filter as a routine check to catch reagents due to expire within a fortnight.

List & search Sorting & pagination

Click the arrows on the D.O.E. column header to sort by expiry — earliest first (▲) or latest first (▼) — so the most urgent reagents rise to the top.

The list shows 50 records per page. The footer tells you how many are shown of the total and offers Prev / Next links, which keep your current filters and sort.

List & search Cloning a record

The Clone action copies an existing preparation into a new blank record — handy for repeat preparations. The clone starts without a reagent number, without verification and without history, so you complete those fresh. Fill in the new number and dates, adjust anything that differs, and save.

PDFs & printing Single-record PDF

The PDF action on a list row (and the PDF button in View mode) generates the controlled PDF for that one record — the formatted preparation log with the Envi Tech AL masthead, chemicals, standardisation and signatures.

PDFs & printing The DRAFT watermark

A single-record PDF for a record that has not yet been verified carries a subtle diagonal DRAFT watermark. Once the record is verified, the watermark disappears and the PDF prints as a final controlled document.

If you print a record and see DRAFT across it, that is your cue the record still needs verifying. Verify it, then re-open the PDF.

PDFs & printing Monthly (stacked) PDF

The Print month button on the list produces a single PDF containing all matching records for one laboratory, stacked like the paper log. Choose Karachi or Lahore (not “All”) and optionally a month, then click Print month.

The monthly print needs a specific lab selected — with the location left on “All” it cannot produce a single-lab log.

Access & safety Access, roles & the control number

This module deliberately gives full access to every logged-in user. There is no role gate blocking who can create, edit, verify or edit the control-document number — the emphasis is on traceability rather than restriction. The system records your name against saves, verifications and standardisation entries.

The per-laboratory control document number (document no., issue no., revision no.) that appears on the printed logs can be maintained by any logged-in user.

Because access is open, treat the audit trail and verification badges as the authoritative record of who did what.

Access & safety Built-in safeguards (summary)

Several protections work quietly in the background to keep records sound:

  • Expiry-before-preparation dates are rejected.
  • Duplicate reagent numbers in the same lab trigger a warning before saving.
  • Concurrent edits are caught so no one silently overwrites another’s changes.
  • Editing a titrant never erases its correction factor.
  • Editing a verified record clears its verification so it is re-checked.
  • An unverified record prints with a DRAFT watermark.
  • Every change is written to the append-only audit trail.

Troubleshooting Common questions

The Standardisation panel isn’t showing

It appears only when the Reagent Type is Standardised titrant. Change the type and it will appear.

My save was blocked

The usual reasons are: the Reagent Name is empty; the expiry date is before the preparation date; or someone else edited the record since you opened it (reload and try again). A duplicate reagent number shows a warning you can confirm past.

There’s a DRAFT watermark on my PDF

The record has not been verified. Verify it, then re-open the PDF for a clean final copy.

The Standardisation History table is empty or missing

It only appears for titrants that have at least one standardisation event logged. Enter the standardisation values and save to create the first entry.

The monthly print won’t generate

Select a single laboratory (Karachi or Lahore) rather than “All”, then use Print month.

My name isn’t on the record

Prepared By defaults to your name on new records; on older or cloned records you may need to set it. Your identity is separately captured in the audit trail whenever you save.
